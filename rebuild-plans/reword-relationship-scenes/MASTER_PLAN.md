## 1. 기획 목적

기존 감정 문해사전은 감정의 이름과 차이를 설명하고, 기존 Re:Word 개별 글은 한 표현이 관계에서 달라지는 의미를 읽는다. 7개 관계언어 지도는 표현을 감정 방어, 대화 조절, 책임 이동, 평가, 통제, 거리 변화, 경계와 회복의 기능으로 분류한다.

새 통합 심화편은 이 자료를 다시 설명하지 않는다. 하나의 실제 관계 상황에서 여러 감정과 여러 표현이 어떻게 만나고, 말한 사람과 들은 사람의 해석이 어디에서 갈리며, 그 차이가 반복될 때 관계가 어떻게 바뀌는지를 분석한다.

핵심 질문은 다음과 같다.

> **이 상황에서 감정·말·해석·관계 변화는 어떻게 연결되며, 서로 다른 상황에서 같은 구조가 왜 반복되는가?**

### 기본 흐름

```text
관계의 장면
→ 겹쳐 있는 감정
→ 실제로 나오는 말과 침묵
→ 말한 사람과 들은 사람의 해석 차이
→ 반복될 때 생기는 관계 변화
→ 다시 말하기·경계·거리 조절
```

### 통합 분석 흐름


## 3. 타이틀 운영 원칙

### H1 기본 형식

```text
[Re:Word 관계의 장면] 상황형 핵심 제목 - 관계문해 질문형 부제
```

### 타이틀 작성 기준

1. 개별 감정어 또는 개별 표현을 제목의 주어로 삼지 않는다.
2. 독자가 실제로 겪는 관계 상황을 앞부분에 둔다.
3. 부제는 감정·말·해석·관계 변화 가운데 핵심 갈등을 드러낸다.
4. 기존 개별 글과 겹치는 `○○ 뜻` 형식은 사용하지 않는다.
5. 한 관계를 악인·피해자 구도로 단정하지 않는다.
6. `왜`, `언제`, `어떻게`, `사이`, `갈리는 지점`을 활용하되 모든 제목을 질문형으로 획일화하지 않는다.
7. 공개 제목에는 내부 관리번호를 넣지 않는다. 관리문서에서만 `RS-01`~`RS-35`를 사용한다.

## 6. 편성 원칙

각 상황군은 **상황 사례형 4편 + 통합 분석형 1편**으로 구성한다.

- 상황 사례형: 구체 장면을 따라 감정·말·해석·관계 변화를 분석
- 통합 분석형: 앞선 4편과 기존 감정·표현 글을 가로질러 반복 구조를 분석
- 상세 설계: [`ARTICLE_PLAN_35.md`](./ARTICLE_PLAN_35.md)

## 7. 상황 사례형 표준 본문 구조

1. **관계의 장면**  
   한 사람의 마음 독백이 아니라 두 사람 이상의 상호작용으로 시작한다.

2. **이 장면에 겹쳐 있는 감정**  
   대표 감정 하나를 확정하지 않고 3~6개의 감정 층위를 제시한다.

3. **실제로 나오는 말과 침묵**  
   기존 Re:Word 개별 표현 여러 편을 하나의 장면 안에 배치한다.

4. **말한 사람과 들은 사람의 해석**  
   의도와 수용이 어긋나는 이유를 비교한다.

5. **이 말이 관계에서 하는 일**  
   7개 기능 지도 가운데 주 기능과 보조 기능을 연결한다.

6. **이 상황이 반복되면 무엇이 달라지는가**  
   한 번의 사건이 오해·의심·체념·역할·거리로 고착되는 시간을 다룬다.

7. **다시 말하기와 거리 조절**  
   무조건 부드러운 말로 바꾸지 않는다. 감정 설명, 대화 보류, 도움 요청, 거절, 경계 설정, 관계 거리 조절 가운데 상황에 맞는 선택지를 제시한다.

8. **함께 읽는 기존 글**  
   감정 글 1~2개, 개별 표현 글 2~4개, 기능 지도 1개를 연결한다.

9. **같은 상황군 이어 읽기**  
   앞 글·다음 글·다섯 번째 통합 분석편으로 이동시킨다.

## 8. 통합 분석형 표준 본문 구조

1. **서로 다른 장면의 공통 질문**
2. **앞선 상황 사례형 네 편 비교**
3. **공통 감정 구조**
4. **표현은 다르지만 반복되는 말의 기능**
5. **해석 충돌을 만드는 조건**
   - 기대의 차이
   - 관계의 역사
   - 말투와 비언어 표현
   - 반복 여부
   - 힘과 지위의 차이
   - 시간과 매체의 차이
6. **관계가 변하는 경로**
7. **상황별 차이와 분석의 한계**
8. **기존 발행글을 가로지르는 연결 지도**
9. **다시 남기는 관계문해 질문**

통합 분석형은 앞선 네 편의 요약문이 아니다. 감정어·표현·기능 지도 여러 편을 교차시켜 서로 다른 상황에서 같은 구조가 반복되는 이유를 설명해야 한다.

- 같은 상황군 앞 글·다음 글
- 상황군의 통합 분석편

### 통합 분석형 하단

- 앞선 사례형 4편 전부
- 감정 문해사전
- 해당 기능 지도
- 다른 상황군 통합 분석편 2개
- 마지막 RS-35에서는 RS-05·10·15·20·25·30을 모두 연결

### 기존 발행글에 추가할 역방향 연결

기존 글 전체를 재작성하지 않는다. 새 글이 발행된 뒤 아래 위치에 짧은 연결 박스만 추가한다.

- 감정 문해사전 목차: `관계 상황에서 이어 읽기`
- 7개 기능 지도: `관계의 장면 심화 읽기`
- 관련 개별 표현 글: `이 말이 여러 감정과 만나는 상황`
- 기존 감정 글: `이 감정이 말과 관계로 이어지는 장면`

확인된 새 발행 주소만 입력하며 `href="#"`, `발행 후 교체`, 빈 주소를 사용하지 않는다.

## 10. 통합 허브 계획

### H1

```text
[Re:Word 관계의 장면] 감정·말·해석·관계 변화는 어떻게 연결되는가
```

### 허브 진입로

1. **관계 상황에서 찾기**  
   7개 상황군과 28개 상황 사례형으로 이동

2. **반복되는 구조에서 찾기**  
   RS-05·10·15·20·25·30·35 통합 분석편으로 이동

3. **감정 이름에서 찾기**  
   기존 감정 문해사전으로 이동

4. **들은 말에서 찾기**  
   7개 관계언어 지도와 기존 개별 표현 글로 이동

### 허브 핵심 인포그래픽

```text
감정 발생
→ 말 또는 침묵 선택
→ 상대의 해석
→ 반응
→ 반복
→ 관계의 거리·역할·경계 변화
```

## 12. 중복 방지 기준

새 글을 작성하기 전에 다음을 확인한다.

- 개별 감정의 뜻풀이가 본문의 중심이 아닌가
- 개별 표현의 숨은 뜻만 다시 설명하고 있지 않은가
- 하나의 표현을 하나의 심리로 단정하지 않는가
- 상황의 시간적 변화와 상호작용이 포함되어 있는가
- 말한 사람과 들은 사람의 해석이 모두 제시되는가
- 기존 감정 글과 표현 글을 실제로 교차 연결하는가
- 통합 분석편에 네 사례의 공통 구조와 차이가 모두 있는가
- 회복을 화해나 관계 유지로만 결론 내리지 않는가

## 14. 확정된 운영 결정

- Re:View는 새 독립 시리즈로 확장하지 않는다.
- 기존 감정 문해사전은 Re:Word의 감정 어휘 근거층으로 흡수한다.
- 신규 통합 심화편의 주 테마는 개별어가 아니라 관계 상황이다.
- 각 상황군에는 반드시 여러 글을 가로지르는 통합 분석편을 둔다.
- 신규 글은 Old-Newbie의 Re:Word 영역에 발행한다.
- 기존 4th-season 및 pencilgon 주소는 삭제·복제하지 않고 연결 자산으로 유지한다.
- 35편의 번호는 내부 관리용이며 공개 제목에는 표시하지 않는다.
- 임시 링크를 만들지 않고 실제 주소가 확인된 뒤 연결한다.
